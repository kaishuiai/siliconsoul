# 业务规划：SiliconSoul

## 1. 定位与价值主张

定位：面向个人与小团队的“可扩展多专家投研助手”，以结构化结果与可追溯链路替代黑盒式回答。

核心价值：

- 多视角：技术面/知识面/决策面并行输出，减少单一模型偏差
- 可解释：每个专家的输入/输出/置信度/耗时可追踪
- 可复盘：落盘存储与批处理能力支持研究迭代
- 可扩展：专家可独立开发、版本化、灰度启停

## 2. 目标用户与需求

目标用户：

- 高频做“问题驱动研究”的个人投资者
- 小型投研团队/研究助理
- 需要集成分析能力的内部工具团队

核心需求：

- 快速得到结论与关键依据
- 更关注“风险点/不确定性/反例”而非单一建议
- 需要一致的输出格式便于比对与归档

## 3. 核心业务指标（North Star & Metrics）

- 北极星：有效分析次数（输出包含可用结论且无关键错误）
- 过程指标：
  - 请求成功率、平均耗时、超时率
  - 单请求专家调用数、聚合一致率
  - 结果被用户采纳/复用次数（可通过保存/导出/复盘行为近似）

## 4. 产品形态与交付路径

阶段 0（当前）：CLI + 本地/容器化运行 + 文档

阶段 1：本机 API 服务（受控内网）+ 简易前端展示

阶段 2：多节点分布式执行（局域网）+ 专家插件化市场/仓库

## 5. 商业化假设（可选）

路径 A：个人订阅

- 免费：基础专家 + 限额分析次数
- 订阅：更多专家、批量分析、存储与导出、策略库

路径 B：团队授权

- 内网部署、权限/审计、定制专家、SLA 与运维支持

## 6. 风险与对策

- 风险：金融合规与误导性建议
  - 对策：明确免责声明；默认只输出分析与不确定性，不执行交易
- 风险：外部数据源不稳定
  - 对策：多源 fallback；缓存；可配置切换
- 风险：架构复杂度过早膨胀
  - 对策：以 MVP 结果链路为主，分布式/前端按里程碑推进
