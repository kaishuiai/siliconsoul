# PRD：SiliconSoul 多专家投资分析系统

## 1. 背景与目标

SiliconSoul 旨在将“股票分析/知识检索/决策支持/执行跟踪”等能力拆解为多个可独立演进的 Expert，并通过编排器（MOE Orchestrator）并行调用与聚合，输出可解释、可追踪的最终结论。

目标：

- 对单条自然语言请求，产出结构化的多专家结果与最终聚合结论
- 支持批量请求与可观测（请求统计、性能、成功率）
- 支持持久化记录（请求、专家结果、聚合结果）与离线复盘
- 支持扩展：新增/移除 Expert、替换存储/分布式执行/实时行情源

非目标（当前阶段）：

- 面向公网的生产级 HTTP 网关（当前代码包含路由器组件，但未包含完整 Web Server 启动与适配层）
- 金融建议合规、自动下单等高风险能力默认关闭

## 2. 用户与使用场景

用户角色：

- 个人投资者：对单票/组合做技术面与事件面快速分析
- 研究/投研：需要批量分析、可追溯的专家输出、可复盘的数据落盘
- 系统集成者：希望以 API/SDK 方式调用（现阶段以 CLI 为主）

典型场景：

- 询问某标的近期趋势、关键指标（MA/RSI/MACD）与风险点
- 查询知识型问题（指标解释、策略原则、风险管理）
- 批量输入多个问题，输出一致格式的结果集合
- 运行期查看系统健康、专家列表、近一段时间的成功率/耗时

## 3. 核心用户旅程

### 3.1 单次分析（CLI）

1. 用户输入问题（文本 + 可选上下文/参数）
2. 编排器选择并发执行 N 个专家（超时控制）
3. 聚合器对专家结果做一致性判断与最终输出
4. 输出结构化结果（专家列表、各自结论、最终结论、置信度/耗时）
5. 可选：写入存储（memory/json/sqlite）

### 3.2 批量分析（CLI）

1. 用户传入多条请求
2. 系统并发调度并聚合每条请求
3. 输出批量结果，并可做统计与落盘

## 4. 功能需求（Functional Requirements）

### 4.1 编排与聚合

- FR-ORCH-01：支持注册/注销 Expert
- FR-ORCH-02：对每个请求并行执行多个 Expert，具备全局超时
- FR-ORCH-03：对每个 Expert 结果记录耗时、错误与置信度
- FR-ORCH-04：输出 AggregatedResult，包含最终结果与专家明细

### 4.2 专家系统

- FR-EXP-01：统一接口 analyze(request) -> ExpertResult（异步）
- FR-EXP-02：支持任务类型声明（get_supported_tasks）
- FR-EXP-03：异常隔离：单个专家失败不影响整体输出（聚合层降级）

### 4.3 数据与存储

- FR-STORE-01：支持 memory/json/sqlite 三种存储后端
- FR-STORE-02：记录请求、专家结果、聚合结果，支持查询与统计

### 4.4 监控与健康

- FR-MON-01：提供健康状态（health/status）
- FR-MON-02：提供窗口统计指标（成功率、平均耗时、吞吐等）

### 4.5 实时行情（可选）

- FR-DATA-01：支持 WebSocket 订阅行情、断线重连与回调
- FR-DATA-02：支持 fallback 到 HTTP 轮询与缓存

### 4.6 API/前端（规划项）

- FR-API-01：提供 /api/health、/api/experts、/api/process、/api/batch 等端点
- FR-FE-01：提供简单 Web UI 展示结果与图表

## 5. 非功能需求（Non-Functional Requirements）

- NFR-01：可扩展性：新增 Expert 不应修改核心编排逻辑
- NFR-02：可靠性：请求超时、单专家错误可被隔离并返回结构化错误
- NFR-03：可观测性：必须可追踪每次请求、各专家耗时与错误
- NFR-04：可配置性：关键参数（超时、并发、存储、数据源）应集中配置，避免硬编码
- NFR-05：可测试性：核心编排、关键专家、存储与数据源应有单测/集成测试

## 6. MVP 范围与验收

MVP（必须）：

- CLI 单次/批量处理可用
- MOE 编排器并发执行、聚合输出结构化结果
- 基础专家：股票分析、知识检索、对话（至少可跑通一条链路）
- 监控与存储至少各支持一种后端

验收标准：

- 对 10 条输入的批量处理可完成并输出结构化结果
- 每条请求都有：专家明细、最终结论、耗时/错误字段
- 监控可读到：请求总数、成功率、平均耗时

## 7. 当前实现状态（简表）

- 编排器、专家抽象、模型契约、存储、监控、实时行情组件、CLI 入口
