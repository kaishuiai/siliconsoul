# 迭代路线图

## 里程碑 M0：可用核心链路（当前）

- CLI 可运行：单次/批量处理
- 编排器并发执行 + 聚合输出结构化结果
- 至少 3 个核心 Expert：股票分析 / 知识检索 / 对话
- 基础存储与监控可用（至少一种后端）

## 里程碑 M1：接口化与可视化

- 提供可启动的 HTTP 服务（内网）
- API 契约稳定（/health、/experts、/process、/batch）
- 前端可展示：请求历史、专家明细、图表

## 里程碑 M2：可复盘与策略化

- 结果落盘与导出（时间范围、标的维度）
- 引入策略库/规则库（可版本化）
- 增强一致性评估：冲突解释、反例提示

## 里程碑 M3：分布式与插件生态（可选）

- 多节点调度与健康检查
- Expert 插件化：独立包/仓库、自动发现、灰度启停
